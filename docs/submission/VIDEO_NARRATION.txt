The demonstration task is a password-reset flow. Codex used G P T five point six Terra to find a stalled audit path, repair it, and pass sixteen deterministic tests. Flight Recorder then bound the final capture to one clean commit, one passing post-commit test, and four committed artifacts.

A fifth synthesises their findings. Every finding points to stable evidence. The model advises. Deterministic logic alone decides whether the passport is ready to seal.

This passport contains five signed review receipts and twenty-two findings. Eighteen were resolved. Four low-severity warnings remain visible as explicitly accepted demonstration-scope risks, not hidden production capabilities.

The browser verifier recomputes every Secure Hash Algorithm 256-bit artifact hash, the hash-linked event chain, the deterministic Merkle root, and the E D two five five one nine signature. It needs no Flight Recorder server, Codex account, or OpenAI Application Programming Interface key.

The claim is deliberately narrow. The covered evidence has not changed since sealing by the corresponding key holder. That does not prove correctness, certified security, signer identity, legal compliance, or trusted time.

Now I change one covered artifact, safely, in browser memory. The signed manifest remains intact, but the artifact no longer matches. Verification turns red immediately, identifies the exact mismatch, and uploads nothing.

Restore the original artifact, and the passport verifies again. That green-to-red-to-green transition is the point. A pull request shows the code that landed. Flight Recorder adds the reviewable evidence around it, and gives any reviewer the means to verify it independently.
