# Flight Recorder Memory Bank Index

The memory bank preserves project continuity. It is not a substitute for live repository, GitHub, browser, deployment, or human-acceptance evidence.

- `projectbrief.md`: stable product mission, users, scope, and claims.
- `activeContext.md`: current verified state, open work, blockers, and the next clean action.
- `../PLANS.md`: execution sequence and delivery gates.
- `../planning/DECISION_LOG.md`: decisions and unresolved decision gates.
- `../planning/BLOCKERS.md`: blockers with owners and minimum next actions.
- `../planning/VALIDATION_MATRIX.md`: requirement-to-evidence status.
- `../CANONICAL_TRUTH.md`: source precedence and truth separation.

Update `activeContext.md` after material implementation, verification, release, deployment, browser, or operator-review changes.
